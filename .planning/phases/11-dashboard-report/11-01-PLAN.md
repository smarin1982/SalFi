       MODULE-LEVEL: only stdlib imports (os, json, logging). No top-level anthropic/fpdf/kaleido imports.

       --- fetch_comparables(company_name: str, country: str, sector: str = "salud") -> list[str] ---
       - Calls web_search.search_sector_context() with query: f"empresas comparables {sector} {country} {company_name} financiero"
       - Returns list of up to 3 result snippets (strings), empty list on any exception
       - NEVER raises — catches all exceptions, logs warning, returns []

       --- generate_executive_report(kpis: dict, red_flags: list, comparables: list, company: dict) -> str ---
       - company dict keys: name (str), country (str), currency_original (str), fiscal_year (int)
       - Checks os.environ.get("ANTHROPIC_API_KEY") — if missing, returns the string:
         "[Error: ANTHROPIC_API_KEY no configurada. Agrega ANTHROPIC_API_KEY=sk-ant-... al archivo .env]"
       - Lazy imports: `from anthropic import Anthropic`
       - Creates client = Anthropic(timeout=120.0)
       - Builds ONE messages.create() call (NOT 4 separate calls) — single call is faster and simpler
       - model="claude-opus-4-6", max_tokens=4096
       - system prompt: "Eres un analista financiero senior especializado en empresas de salud de Latinoamérica. Genera reportes ejecutivos concisos, factuales y en español profesional. No inventes datos; si un dato falta, indícalo explícitamente."
       - user prompt structure (f-string):
         ```
         Genera un reporte ejecutivo para {company['name']} ({company['country']}, Año fiscal {company['fiscal_year']}) con estas cuatro secciones exactas:

         ## 1. Resumen de Gestión
         ## 2. KPIs Destacados
         ## 3. Red Flags Activas
         ## 4. Contexto Sectorial y Comparables

         Datos disponibles:
         KPIs (valores en USD normalizados): {json.dumps(kpis, ensure_ascii=False, indent=2)}
         Red Flags detectadas: {json.dumps(red_flags, ensure_ascii=False, indent=2)}
         Empresas comparables (búsqueda web): {comparables}
         Moneda original: {company.get('currency_original', 'N/D')}

         Instrucciones por sección:
         - Sección 1: 2-3 párrafos de evaluación general de la gestión financiera
         - Sección 2: tabla markdown de los 5 KPIs más relevantes con valor + interpretación; muestra valores tanto en moneda original como en USD (ej: "COP 4.2B (~USD 1.05M)")
         - Sección 3: lista numerada; para cada red flag indica severidad (Alta/Media/Baja) y una oración de contexto
         - Sección 4: menciona 2-3 empresas comparables del sector con benchmarks sectoriales disponibles; si no hay comparables, indica "No se encontraron comparables disponibles."
         ```
       - Returns msg.content[0].text
       - On anthropic.APIError or any exception: logs error, returns "[Error al generar reporte: {str(e)}]"

       --- build_pdf_bytes(report_text: str, company_name: str, country: str, fiscal_year: int) -> bytes ---
       - Lazy imports: `from fpdf import FPDF`
       - Creates FPDF instance, set_auto_page_break(auto=True, margin=15)
       - add_page()
       - Header section (set_font "Helvetica" "B" 16): cell with text f"Reporte Ejecutivo — {company_name}"
       - Sub-header (set_font "Helvetica" "" 11): cell with f"{country} · Año fiscal {fiscal_year}"
       - Sub-header (set_font "Helvetica" "" 9, text color RGB 128,128,128): cell with f"Generado: {datetime.now().strftime('%Y-%m-%d')}"
       - ln(8) separator
       - set_font "Helvetica" "" 10
       - Use multi_cell(0, 5, report_text) to render the full markdown text
         NOTE: fpdf2 multi_cell does NOT render markdown formatting — pound signs and asterisks appear as literals.
         This is acceptable for the initial version; report_text is professional prose that reads fine as plain text.
       - Returns bytes(pdf.output())
       - Raises RuntimeError (with install hint) if fpdf ImportError occurs

    CRITICAL: All Spanish characters (á é í ó ú ñ ü ¿ ¡) are within Latin-1 — Helvetica handles them.
    Do NOT add DejaVu/TTF font handling unless Helvetica fails in testing.
