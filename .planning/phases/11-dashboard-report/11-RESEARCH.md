## Summary

Phase 11 is the final integration layer of the v2.0 LATAM pipeline. It adds a dedicated LATAM section to the existing `app.py` Streamlit dashboard. The critical architectural constraint is that this must be **purely additive** — the S&P 500 section must be unaffected. All LATAM package imports must be lazy (inside functions with `try/except ImportError`), and all new widget keys must carry the `latam_` prefix.

The phase has two major technical deliverables: (1) the Streamlit LATAM section UI, and (2) the executive PDF report generated by Claude API. The PDF library decision is now resolved: **use fpdf2** (pure Python, no system dependencies, Streamlit-native pattern). WeasyPrint's GTK3/MSYS2 dependency was flagged as a spike in STATE.md; research confirms it remains high-risk on Windows and should be rejected in favor of fpdf2.

For Plotly chart export (embedding charts in the PDF), **Kaleido 1.0** requires Chrome or Chromium to be installed — which aligns with the existing Playwright setup (Playwright also installed Chromium). Charts are exported as PNG bytes via `fig.to_image(format="png")` and embedded in the fpdf2 document.

**Primary recommendation:** fpdf2 for PDF generation (no system deps), Anthropic SDK synchronous `client.messages.create()` for Claude API, `st.tabs` to partition S&P 500 vs. LATAM sections (cleanest isolation), Kaleido for PNG chart export leveraging existing Playwright Chromium install.

### Pattern 1: st.tabs for Section Isolation (DASHL-03)

**What:** Wrap the existing S&P 500 UI and the new LATAM UI in `st.tabs`. The existing code moves into `with tab_sp500:` unchanged.

**When to use:** When adding a major new feature section to an existing Streamlit app where complete visual separation is desired.

**Why tabs over expanders:** Expanders share the same page scroll space; a collapsed LATAM expander still renders all widgets (causing potential DuplicateWidgetID). Tabs fully partition the rendering context and are visually cleaner.

**Critical:** All content in all tabs is computed on every rerun. Do not put slow operations (Claude API call, PDF generation) at tab-render level — gate them behind buttons.

```python
# Source: Official Streamlit docs - st.tabs
tab_sp500, tab_latam = st.tabs(["S&P 500", "LATAM"])

with tab_sp500:
    # paste all existing app.py content here — zero changes
    pass

with tab_latam:
    init_latam_session_state()
    render_latam_section()
```

### Pattern 2: Lazy Imports for LATAM Packages (DASHL-03, Pitfall 11)

**What:** Every LATAM-specific import lives inside the function that uses it, never at module top level.

**When to use:** For any optional feature whose dependencies may fail to import (missing GTK, Playwright browsers not installed, etc.).

```python
# CORRECT — lazy import pattern
def generate_pdf_report(report_data: dict) -> bytes:
    try:
        from fpdf2 import FPDF
    except ImportError:
        raise RuntimeError(
            "fpdf2 not installed. Run: pip install fpdf2"
        )
    pdf = FPDF()
    # ... build PDF ...
    return bytes(pdf.output())

def call_claude_api(prompt: str, system: str) -> str:
    try:
        from anthropic import Anthropic
    except ImportError:
        raise RuntimeError(
            "anthropic SDK not installed. Run: pip install anthropic"
        )
    client = Anthropic()  # reads ANTHROPIC_API_KEY from env
    msg = client.messages.create(
        model="claude-opus-4-6",
        max_tokens=4096,
        system=system,
        messages=[{"role": "user", "content": prompt}],
    )
    return msg.content[0].text

def export_chart_png(fig) -> bytes:
    try:
        import kaleido  # noqa: F401
    except ImportError:
        raise RuntimeError("kaleido not installed. Run: pip install kaleido")
    return fig.to_image(format="png", width=800, height=300, scale=2)
```

### Pattern 3: Currency Toggle (FX-03)

**What:** `st.radio` or `st.segmented_control` with "Moneda Original" / "USD" options. Stored in session state. KPI card render functions accept `currency_mode` parameter and use it to select the correct value column.

**Why:** The Parquet files store USD-normalized values. The `meta.json` stores `currency_original` (e.g., "COP"). Displaying original-currency values requires reverse-FX: `usd_value / fx_rate`. Store the fx_rate in `meta.json` (already part of extraction metadata from Phase 6).

### Pattern 5: Claude API Report Generation (RPT-01, RPT-03)

**What:** The executive report is generated by calling `client.messages.create()` once per section (4 calls total) or once for the full report with a structured system prompt. Gate the call behind a button; cache result in `st.session_state` so Streamlit reruns don't re-call the API.

**Warning:** Claude API calls can take 10-60 seconds for a full 4-section report. Use `st.spinner()`. Store result in session state — do NOT call inside a bare tab render path.

```python
# Source: Anthropic Python SDK official docs
def generate_executive_report(kpis: dict, red_flags: list, comparables: list, company: dict) -> str:
    try:
        from anthropic import Anthropic
    except ImportError:
        return "[Anthropic SDK not installed]"

    client = Anthropic()  # uses ANTHROPIC_API_KEY env var

    system = (
        "Eres un analista financiero senior especializado en empresas de salud de Latinoamérica. "
        "Genera reportes ejecutivos concisos, factuales y basados en los datos proporcionados. "
        "Usa formato profesional en español. No inventes datos; si un dato no está disponible, indícalo."
    )

    user_prompt = f"""
Genera un reporte ejecutivo para {company['name']} ({company['country']}) con las siguientes secciones:

## 1. Resumen de Gestión
## 2. KPIs Destacados
## 3. Red Flags Activas
## 4. Contexto Sectorial y Comparables

Datos disponibles:
- KPIs: {kpis}
- Red Flags: {red_flags}
- Empresas comparables (de búsqueda web): {comparables}
- Moneda original: {company.get('currency_original', 'N/D')}

Instrucciones:
- Sección 1: 2-3 párrafos de evaluación general
- Sección 2: tabla de 5 KPIs más relevantes con interpretación
- Sección 3: lista numerada de red flags con severidad y contexto
- Sección 4: 2-3 comparables mencionados con benchmarks sectoriales
"""

    msg = client.messages.create(
        model="claude-opus-4-6",
        max_tokens=4096,
        system=system,
        messages=[{"role": "user", "content": user_prompt}],
    )
    return msg.content[0].text
```

### Pattern 6: fpdf2 PDF Generation + st.download_button (RPT-02)

**What:** Use fpdf2 to generate a PDF from the report text. Return as `bytes`. Pass to `st.download_button` with `mime="application/pdf"`.

```python
# Source: fpdf2 official docs — Usage in Web APIs section
def build_pdf_from_report(report_text: str, company_name: str, chart_pngs: list[bytes]) -> bytes:
    try:
        from fpdf import FPDF
    except ImportError:
        raise RuntimeError("fpdf2 not installed.")

    pdf = FPDF()
    pdf.set_auto_page_break(auto=True, margin=15)
    pdf.add_page()

    # Header
    pdf.set_font("Helvetica", "B", 16)
    pdf.cell(0, 10, f"Reporte Ejecutivo — {company_name}", ln=True)
    pdf.ln(5)

    # Report text (multi_cell handles line wrapping)
    pdf.set_font("Helvetica", size=10)
    pdf.multi_cell(0, 5, report_text)

    # Chart PNGs (from kaleido)
    for png_bytes in chart_pngs:
        pdf.add_page()
        # Write PNG bytes to temp file for fpdf2 image embedding
        import tempfile, os
        with tempfile.NamedTemporaryFile(suffix=".png", delete=False) as tmp:
            tmp.write(png_bytes)
            tmp_path = tmp.name
        try:
            pdf.image(tmp_path, x=10, w=190)
        finally:
            os.unlink(tmp_path)

    return bytes(pdf.output())

# In Streamlit — gate behind button, cache in session_state
if st.button("Generar Reporte", key="latam_generate_report"):
    with st.spinner("Generando reporte ejecutivo con Claude API..."):
        st.session_state["latam_report_text"] = generate_executive_report(...)
        st.session_state["latam_report_pdf"] = build_pdf_from_report(...)

if "latam_report_pdf" in st.session_state:
    st.download_button(
        label="Descargar PDF",
        data=st.session_state["latam_report_pdf"],
        file_name=f"reporte_{company_slug}.pdf",
        mime="application/pdf",
        key="latam_download_pdf",
    )
```

### Anti-Patterns to Avoid

- **Top-level LATAM imports in app.py:** `import weasyprint` or `import anthropic` at module top level. If the GTK DLL fails or the API key is missing, the entire app crashes. Always lazy-import.
- **Claude API call on every Streamlit rerun:** `generate_executive_report()` called outside a button guard. Every widget interaction triggers a full rerun and re-invokes the expensive API call. Gate behind `st.button` and cache result in `st.session_state`.
- **st.tabs content at module level:** Putting slow operations (Claude API, Kaleido PNG export) directly in the `with tab_latam:` block outside any button. All tab content runs on every rerun.
- **DuplicateWidgetID:** Any new widget without a `latam_` prefix key. The existing S&P 500 section uses keys like `new_ticker_input`, `load_ticker_btn`, `kpi_{group_name}`. New widgets must use `latam_` prefix.
- **Modifying existing S&P 500 functions:** `render_kpi_card`, `build_trend_figure`, `format_kpi`, `load_kpis`, `get_available_tickers` — none of these should be touched. Create parallel LATAM versions.

## Don't Hand-Roll

| Problem | Don't Build | Use Instead | Why |
|---------|-------------|-------------|-----|
| PDF generation | Custom HTML+CSS PDF builder | `fpdf2` | Handles page breaks, fonts, image embedding, table layout — edge cases in manual PDF generation are numerous |
| PNG from Plotly figure | Canvas rendering, screenshot | `kaleido` + `fig.to_image()` | Kaleido handles the headless rendering pipeline |
| Claude API client | `requests` to Anthropic API manually | `anthropic` SDK | SDK handles auth, retries, timeout, rate limiting, type safety |
| Currency display toggle | Custom JS widget | `st.radio(horizontal=True)` | Streamlit's native horizontal radio is sufficient |
| Download button | Custom HTML anchor with base64 data URL | `st.download_button` | Native, handles MIME type, browser compatibility |

**Key insight:** The PDF generation stack is deceptively complex if hand-rolled — character encoding, page breaks at word boundaries, image DPI, table overflow. fpdf2 handles all of this.

## Common Pitfalls

### Pitfall 1: WeasyPrint GTK DLL Load Failure (RESOLVED — use fpdf2 instead)

**What goes wrong:** `OSError: cannot load library 'gobject-2.0': error 0x7e` on first `weasyprint.HTML().write_pdf()` call. This error only appears at write time, not at import time — meaning a superficial smoke test of `import weasyprint` passes but actual PDF generation fails.

**Why it happens:** WeasyPrint requires GTK3 system DLLs (Pango, Cairo, GDK-PixBuf) installed via MSYS2. If `C:\msys64\mingw64\bin` is not on PATH or `WEASYPRINT_DLL_DIRECTORIES` is not set, the runtime DLL load fails.

**Resolution:** Use fpdf2 instead. STATE.md explicitly flagged WeasyPrint as a spike. Research confirms the risk is real. fpdf2 is pure Python with no system dependencies. Switch from WeasyPrint to fpdf2 in the plan — do not attempt WeasyPrint in Phase 11.

**Warning signs:** `OSError: cannot load library` referencing `gobject`, `cairo`, `pango`, or `gdk_pixbuf`

### Pitfall 2: Kaleido 1.0 Requires Chrome (Not Bundled)

**What goes wrong:** `kaleido.chromium.CalledProcessError` or silent empty bytes from `fig.to_image()` if Chrome is not found.

**Why it happens:** Kaleido versions prior to 1.0 bundled their own Chromium. Kaleido 1.0+ requires an existing Chrome/Chromium installation. The Playwright Chromium downloaded in Phase 6 may or may not be auto-discovered by Kaleido.

**How to avoid:** In Wave 0, run:
```python
import kaleido
kaleido.get_chrome_sync()  # downloads Chrome if not found
# OR verify existing Chrome at default locations
```
Kaleido checks `C:\Program Files\Google\Chrome\Application\chrome.exe` and user AppData. Playwright stores Chromium in `%LOCALAPPDATA%\ms-playwright\` — Kaleido may NOT auto-discover this path. If kaleido cannot find Chrome, call `kaleido.get_chrome_sync()` once during app startup.

**Warning signs:** `to_image()` returns empty bytes or raises subprocess error; Kaleido log shows "Chrome not found"

### Pitfall 3: Claude API Call Timeout in Streamlit (RPT-01)

**What goes wrong:** A 4-section executive report generation can take 30-90 seconds with `claude-opus-4-6`. Streamlit's default request timeout may cut off the response. Users see a spinner that never resolves.

**Why it happens:** The Anthropic SDK default timeout is 10 minutes, but network proxies and Streamlit's connection handling can interrupt long-running HTTP responses. Additionally, if the user navigates away or reloads, the session state with the cached result is lost.

**How to avoid:**
```python
from anthropic import Anthropic
client = Anthropic(timeout=120.0)  # explicit 2-minute timeout
```
Use `stream=True` with Streamlit's `st.write_stream()` for real-time streaming output — this gives visual feedback and prevents timeout. Cache result in `st.session_state` so it survives reruns.

**Warning signs:** Spinner runs > 60 seconds and never completes; `APITimeoutError` in logs

### Pitfall 4: DuplicateWidgetID When Moving Existing Code into st.tabs

**What goes wrong:** The existing S&P 500 code uses widget keys like `kpi_Crecimiento`, `new_ticker_input`, `load_ticker_btn`. Moving this code into `with tab_sp500:` preserves all existing keys. Any new LATAM widget that accidentally reuses these names causes `DuplicateWidgetID`.

**How to avoid:**
1. All new LATAM widgets use `latam_` prefix (already required by STATE.md decisions)
2. Audit: after adding LATAM section, grep `app.py` for duplicate `key=` values
3. The year-range slider in the sidebar uses `label_visibility="collapsed"` without an explicit `key=` — Streamlit derives the key from the label. This is safe as long as the label is unique.

**Warning signs:** `streamlit.errors.DuplicateWidgetID` on startup; error message names a key you didn't just add

### Pitfall 5: st.tabs All-Content-Computed Issue

**What goes wrong:** Placing slow operations (LatamAgent.run(), Claude API call) directly in the `with tab_latam:` block causes them to execute on every Streamlit rerun — even when the user is viewing the S&P 500 tab. This makes the S&P 500 tab slow.

**Why it happens:** Streamlit computes all tab content on every script execution. Content is not conditionally rendered based on which tab is active.

**How to avoid:** Gate all slow operations behind buttons. Store results in `st.session_state`. The tab block should only render cached data, not trigger computation.

```python
with tab_latam:
    if st.button("Ejecutar Análisis LATAM", key="latam_run_btn"):
        with st.spinner("Ejecutando pipeline LATAM..."):
            # slow operation here — only runs when button clicked
            run_latam_pipeline()
    # fast display of cached results
    if "latam_results" in st.session_state:
        render_latam_results()
```

### Pitfall 6: fpdf2 Latin-1 Encoding for Spanish Text

**What goes wrong:** Company names and report text containing Spanish characters (`ñ`, `á`, `é`, `í`, `ó`, `ú`) display as garbled text or raise `UnicodeEncodeError` in fpdf2 when using the default Helvetica font (which is Latin-1 encoded).

**Why it happens:** fpdf2's built-in fonts (Helvetica, Times, Courier) are Latin-1. Characters outside the Latin-1 range (e.g., some Unicode characters) fail. Spanish characters within Latin-1 (accented vowels, ñ) work but only if the string is not double-encoded.

**How to avoid:** Use a Unicode-capable font. fpdf2 supports True Type fonts:
```python
pdf = FPDF()
# Add a TTF font that supports Spanish characters
# Option 1: Use fpdf2's built-in core fonts (covers Latin-1 — works for Spanish)
pdf.set_font("Helvetica", size=10)
# Spanish chars á é í ó ú ñ ü ¿ ¡ are within Latin-1 — this works

# Option 2: For full Unicode (if needed):
# pdf.add_font("DejaVu", fname="DejaVuSans.ttf", uni=True)
# pdf.set_font("DejaVu", size=10)
```

For a financial report with standard Spanish text, Helvetica (Latin-1) covers all needed characters. Only needed if company names contain characters outside Latin-1.

### Pitfall 7: ANTHROPIC_API_KEY Not Available in App Environment

**What goes wrong:** `anthropic.AuthenticationError: 401 Unauthorized` when `ANTHROPIC_API_KEY` is not set in the environment where Streamlit runs.

**How to avoid:**
```python
# Check at report generation time, not at import time
import os
def generate_executive_report(...) -> str:
    if not os.environ.get("ANTHROPIC_API_KEY"):
        return "[Error: ANTHROPIC_API_KEY no configurada. Agrega la clave al archivo .env]"
    # proceed with API call
```
The project uses `.env` (evidenced by `M .env` in git status). Streamlit loads `.env` via `python-dotenv` if configured. Verify `.env` contains `ANTHROPIC_API_KEY=sk-ant-...`.

## State of the Art

| Old Approach | Current Approach | When Changed | Impact |
|--------------|------------------|--------------|--------|
| WeasyPrint + GTK3 Runtime Windows installer | MSYS2 for GTK3 DLLs | WeasyPrint v52+ (2022) | Old installer discontinued; MSYS2 is now the only supported Windows path |
| Kaleido 0.x (bundled Chromium) | Kaleido 1.0 (requires system Chrome) | Kaleido 1.0.0 (Nov 2025) | Must have Chrome installed separately; `kaleido_get_chrome` CLI for CI |
| `duckduckgo-search` package | `ddgs` package | 2024 | Same author, renamed — old package deprecated |
| `client.messages.create()` with engine="..." | engine parameter deprecated | Plotly 6.1 (2025) | Do not pass `engine=` to `write_image()`; it's deprecated and removed after Sep 2025 |

**Deprecated/outdated:**
- `fig.write_image(engine="kaleido")` — the `engine` parameter is deprecated in Plotly 6.1, removed after September 2025. Use `fig.to_image()` directly.
- `duckduckgo-search` pip package — use `ddgs` instead (same API, different package name)

## Open Questions

1. **Kaleido Chrome Discovery**
   - What we know: Kaleido 1.0 looks for Chrome at standard Windows locations (`C:\Program Files\Google\Chrome\...`) and user AppData. Playwright installs Chromium in `%LOCALAPPDATA%\ms-playwright\`.
   - What's unclear: Whether Kaleido auto-discovers the Playwright Chromium path on this specific machine, or whether `kaleido.get_chrome_sync()` is needed as a setup step.
   - Recommendation: In Plan 11-01, make the first task a Wave 0 smoke test: `fig.to_image(format="png")` on a simple figure. If it fails, run `kaleido.get_chrome_sync()` and document in requirements.

2. **fpdf2 vs ReportLab for Tables**
   - What we know: fpdf2 supports tables (HTML-style table API in fpdf2 2.7+) and PNG image embedding. ReportLab's table API is more mature for complex multi-column layouts.
   - What's unclear: Whether the 4-section executive report needs complex table layout that fpdf2 cannot handle.
   - Recommendation: Use fpdf2 first. The report is primarily narrative text with simple KPI tables. If table complexity requires ReportLab, the switch is isolated to one function.

3. **Claude API Response Time vs. Streamlit UX**
   - What we know: `claude-opus-4-6` can take 20-60 seconds for 4096 token responses. `st.spinner()` works for this duration. Streaming via `stream=True` provides progressive display.
   - What's unclear: Whether streaming via `st.write_stream()` integrates cleanly with Streamlit >=1.35 on this machine.
   - Recommendation: Default to synchronous call with `st.spinner()`. Add streaming as a stretch goal if UX feels sluggish during testing.

## PDF Library Decision Record

**Decision: Use fpdf2 (not WeasyPrint)**

Evidence:
1. STATE.md explicitly says WeasyPrint is a spike — "decision finalizes before any template work; fallback is reportlab or fpdf2"
2. PITFALLS.md Pitfall 4 documents the exact failure mode: `OSError: cannot load library 'gobject-2.0'`
3. WeasyPrint MSYS2 requirement is confirmed in official WeasyPrint docs and GitHub issues as the ONLY supported Windows path as of v52+
4. fpdf2 is pure Python, no system dependencies, Streamlit integration officially documented
5. Executive report is text-heavy with simple tables — fpdf2 is sufficient

**Chosen alternative:** fpdf2 >= 2.8.7
