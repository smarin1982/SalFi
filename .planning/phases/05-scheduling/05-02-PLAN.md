<task type="auto">
  <name>Task 1: Register the task with Windows Task Scheduler via schtasks CLI</name>
  <files>.planning/phases/05-scheduling/quarterly_etl_task.xml</files>
  <action>
Run the registration command to import the XML task definition into Windows Task Scheduler. This is done via Bash (schtasks.exe is available from bash on Windows 11):

```bash
schtasks /create /XML "C:\\Users\\Seb\\AI 2026\\.planning\\phases\\05-scheduling\\quarterly_etl_task.xml" /TN "AI2026_QuarterlyETL" /F
```

The `/F` flag forces overwrite if the task already exists (idempotent, safe to re-run).

If schtasks returns a non-zero exit code:
- Exit 5 (access denied): The task still registered — this is a spurious UAC warning on Windows 11 for InteractiveToken tasks. Verify with `schtasks /query /TN "AI2026_QuarterlyETL"` — if the task appears, registration succeeded.
- "Element not found" or "The system cannot find the file": The XML path is wrong. Verify the file exists at the path above.
- "The task XML is malformed": The XML has an encoding issue. Open quarterly_etl_task.xml and ensure the `<?xml version="1.0" encoding="UTF-16"?>` declaration is present.

After registration, verify the task appears:
```bash
schtasks /query /TN "AI2026_QuarterlyETL" /FO LIST
```

Expected output includes:
- `TaskName: \AI2026_QuarterlyETL`
- `Status: Ready`
- `Next Run Time: 4/1/2026 6:00:00 AM` (or similar quarterly date)

Do NOT proceed to Task 2 if the task does not appear in schtasks /query output.
  </action>
  <verify>
    <automated>schtasks /query /TN "AI2026_QuarterlyETL" /FO LIST 2>&1 | grep -i "TaskName\|Status\|Next Run"</automated>
    <manual>Output must show TaskName: \AI2026_QuarterlyETL and Status: Ready</manual>
  </verify>
  <done>schtasks /query /TN "AI2026_QuarterlyETL" shows the task with Status: Ready and a Next Run Time in a future quarter.</done>
</task>

```bash

sleep 30
schtasks /query /TN "AI2026_QuarterlyETL" /FO LIST /V 2>&1 | grep -i "Last Run\|Status\|Result"
```

If Status shows "Running", wait longer and re-check. Do not kill the task.

Once completed, find and display the log:
```bash

cat "C:/Users/Seb/AI 2026/logs/"etl_*.log 2>/dev/null | tail -50
```

Expected log content:
- Line starting with `[YYYYMMDD] Starting quarterly ETL`
- Loguru output from agent.py showing ticker processing (e.g., `INFO | agent:run_batch | Processing AAPL...`)
- Line showing `Batch complete — success: N, skipped: N, failed: 0` (or failed > 0 is acceptable if some tickers have API issues, but the scheduler itself worked)

- Check that scheduler.bat contains `IF NOT EXIST "%LOGDIR%" MKDIR "%LOGDIR%"` — if missing, fix scheduler.bat by adding the MKDIR line, then re-run.

- The PYTHON path in scheduler.bat is wrong. Verify: `ls -la "C:/Users/Seb/miniconda3/python.exe"` — if missing, find the actual path with `which python` from an activated conda shell and update scheduler.bat.
