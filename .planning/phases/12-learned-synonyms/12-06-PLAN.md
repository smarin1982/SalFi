# Plan 12-06 — Smart Web Scraper with Per-Provider Learning

## Objective
Diagnose why DDGS search returns wrong PDFs (e.g., Mexican government report instead of
MiRed IPS financials), then build an adaptive scraper that:
1. Evaluates the full corporate URL before searching externally
2. Learns each provider's document structure (URL patterns, page paths, link texts)
3. Persists learned profiles in `data/latam/scraper_profiles.json` for reuse

---

## Root Cause Analysis (known from session)

Current `search_and_download()` uses DDGS with generic queries like:
  `"mired-ips estados financieros 2024 filetype:pdf"`

Problems:
- DDGS returns the first PDF it finds with those keywords — often a wrong document
  (found a Mexican INEGI government report for a Colombian IPS query)
- No validation that the PDF domain matches the company's domain
- No crawling of the actual corporate website before resorting to external search
- No memory of where this provider published their reports last time

---

## Tasks

### SCRAPE-01 — Diagnose and score current search results

Add `_validate_pdf_relevance(pdf_url: str, company_domain: str, company_name: str) -> float`:
- Returns score 0.0–1.0 based on:
  - Domain match: pdf_url domain == company_domain → +0.5
  - Keywords in URL path: "estados-financieros", "informe", "reporte", "financiero" → +0.2
  - Year in URL: current or prior year → +0.1
  - PDF size > 100KB (scanned financials are large) → +0.1
  - Filetype is PDF → +0.1
- Only download if score >= 0.5; else try next result or next strategy

### SCRAPE-02 — Crawl corporate website before external search

Add `_crawl_corporate_site(domain: str, slug: str) -> Optional[str]` using Playwright:

```
Strategy order:
1. Visit domain root → look for links containing: "financiero", "informe", "estados",
   "reporte anual", "transparencia", "rendicion de cuentas", "documentos"
2. Follow those links → look for direct PDF links or embedded PDF viewers
3. Check common paths: /informes/, /transparencia/, /documentos/, /reportes/,
   /estados-financieros/, /sala-de-prensa/
4. If PDF found on the corporate domain → download it (highest trust)
5. Only if corporate crawl fails → fall back to DDGS
```

### SCRAPE-03 — Learned scraper profiles (per provider)

`data/latam/scraper_profiles.json` — persists across runs:

```json
{
  "mired-ips": {
    "domain": "miredbarranquilla.com",
    "last_success": "2026-03-10",
    "strategy": "corporate_crawl",
    "pdf_url_pattern": "/documentos/estados-financieros-*.pdf",
    "nav_path": ["Transparencia", "Informes Financieros"],
    "failed_ddgs_queries": ["xxxi_2025_ef_t1"],
    "notes": "PDFs en sección Transparencia > Informes"
  }
}
```

On each run:
- Load profile for this slug (if exists)
- Try `pdf_url_pattern` first (direct URL with year substitution)
- If fails, use `nav_path` to navigate the site
- On success, update profile with new URL pattern
- On DDGS success, record the query that worked AND validate domain matches

New function `_save_scraper_profile(slug, profile_update)` — append-only updates,
never overwrites successful patterns.
