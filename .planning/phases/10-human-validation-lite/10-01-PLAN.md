<objective>
Implement the Human Validation Lite gate: a Streamlit form panel that intercepts the extraction result before any Parquet write, lets the analyst review and correct the four key financial values, and only writes to disk after explicit confirmation.

Purpose: LATAM PDF extraction has inherent uncertainty (OCR, multi-column layouts, Spanish synonym mapping). The validation gate compensates by letting a human catch extraction errors before they corrupt the KPI calculation. No US S&P 500 pipeline logic is modified.

Output:
- `latam_validation.py` — standalone validation module with form renderer, confirm/discard handlers, and meta.json writer
- `app.py` — LATAM validation panel wired into session state (lazy import, no top-level import of LATAM modules)
</objective>

**3. `_handle_discard() -> None`**

**4. `_handle_confirm(extraction_result: dict, company: dict, ingresos: float, utilidad_neta: float, total_activos: float, deuda_total: float) -> None`**

**Baja-confidence guard (must run BEFORE any disk write):**
Build `FIELDS = ["ingresos", "utilidad_neta", "total_activos", "deuda_total"]`. For each field, check if `extraction_result.get(f"confidence_{field}") == "Baja"`. If so, compare the incoming corrected value to the original extracted value (`float(extraction_result.get(field) or 0.0)`). If ANY Baja field still equals its original extracted value (meaning the analyst did not edit it), call `st.error("Debe corregir los campos con confianza Baja antes de confirmar.")` and `return` immediately — do not proceed to disk write. This enforces the locked UX requirement without using `disabled=True` (Streamlit bug #8075).

**After the guard passes:**
Build `corrected_values = {"ingresos": ingresos, "utilidad_neta": utilidad_neta, "total_activos": total_activos, "deuda_total": deuda_total}`.
Build `original_values` from `extraction_result` keys of same names.

**Navigation state (must be captured BEFORE clearing session state):**
Capture `slug = company.get("slug", "")` and `country = company.get("country", "")`.

Wrap the entire disk-write operation in a try/except:
- Inside try: lazy import `latam_processor` with `try/except ImportError` (show `st.error(...)` if missing). Call `latam_processor.process_with_validation(company, corrected_values, extraction_result)`. Call `write_meta_json(...)`. Call `st.cache_data.clear()`. Then set `st.session_state["active_latam_company"] = {"slug": slug, "country": country}` so app.py can navigate to the confirmed company's KPI view on the next rerun. Then delete `st.session_state["latam_pending_extraction"]` and `st.session_state["latam_pending_company"]` (do NOT call `_handle_discard()` here — discard now sets `latam_show_rerun`, which is wrong after a successful confirm). Show `st.success("Datos guardados correctamente.")`. Call `st.rerun()`.
- CRITICAL: Do NOT clear session state keys inside the except block — let the analyst retry on error. Show `st.error(f"Error al guardar: {e}")` in except.
- This preserves the atomic write invariant: session state is cleared only after both Parquet files and meta.json are written successfully.

if "latam_pending_extraction" in st.session_state:
    st.divider()
    st.markdown("### LATAM — Validacion de Extraccion")
    try:
        import latam_validation  # lazy import — S&P 500 section unaffected
        latam_validation.render_latam_validation_panel(
            extraction_result=st.session_state["latam_pending_extraction"],
            company=st.session_state.get("latam_pending_company", {}),

        st.error(f"Error de importacion LATAM: {e}. Instale las dependencias LATAM.")

elif st.session_state.get("latam_show_rerun"):
    # Analyst discarded — show re-run button so they can restart without navigating away
    st.divider()
    st.info("Extraccion descartada. No se escribio ningun dato.")
    if st.button("Volver a extraer", key="latam_rerun_btn"):
        del st.session_state["latam_show_rerun"]

        st.rerun()

elif st.session_state.get("active_latam_company"):
    # Analyst just confirmed — navigate to the confirmed company's KPI view
    _co = st.session_state["active_latam_company"]
    st.divider()

    st.rerun()
