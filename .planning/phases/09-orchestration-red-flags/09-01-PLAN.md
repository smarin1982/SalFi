   - Decorated with `@retry(stop=stop_after_attempt(3), wait=wait_exponential(multiplier=1, min=4, max=30), retry=retry_if_exception_type((RatelimitException, DuckDuckGoSearchException)), reraise=False)`
   - Body: `with DDGS() as ddgs: return ddgs.text(query, max_results=max_results)`

3. Public function `search_sector_context(company_name: str, country: str, sector: str = "salud") -> list[dict]`:
   - Guard: if not `_DDGS_AVAILABLE`, log warning and return `[]`
   - Query: `f"{company_name} sector {sector} {country} comparables financieros"`
   - Call `_search_with_retry(query, max_results=5)` inside try/except Exception
   - On exception: `logger.warning(f"Web search failed (non-blocking): {exc!r}")` and return `[]`
   - On success: `logger.info(...)` and return `results or []`
   - NEVER raises — catch-all except is the non-negotiable safety net

4. Public function `search_comparable_companies(sector: str, country: str, max_results: int = 3) -> list[dict]`:
   - Same guard pattern
   - Query: `f"empresas sector {sector} {country} estados financieros comparables"`
   - Same non-raising pattern

**Step C: Create red_flags.py**

Create `red_flags.py` in the project root with the exact implementation from research (Pattern 4):

1. `RedFlag` dataclass with fields: `flag_id`, `name`, `description`, `severity` (str: "Alta"|"Media"|"Baja"), `kpi` (str|None), `kpi_value` (float|None), `fiscal_year` (int|None), `threshold_triggered` (dict)

2. `load_config(config_path: Path = CONFIG_PATH) -> dict`:
   - If `config_path` does not exist: `logger.warning("red_flags.yaml not found — skipping flag evaluation"); return {}`
   - Otherwise: `with open(config_path, "r", encoding="utf-8") as f: return yaml.safe_load(f)`
   - Use `yaml.safe_load()` — NEVER `yaml.load()` (security risk)

3. `_evaluate_threshold(value: float, threshold: dict) -> bool`:
   - Check keys: "gt", "lt", "gte", "lte" — all present conditions must be True

4. `_evaluate_special_flags(kpis_sorted, financials_sorted, config) -> list[RedFlag]`:
   - FLAG-S01: check `operating_cash_flow` in financials_sorted columns; latest row; if fcf < 0 AND net_income > 0 → Alta severity
   - FLAG-S02: iterate net_profit_margin column sorted by fiscal_year; track consecutive negatives; >= 2 → Alta, == 1 → Media
   - Handles missing columns gracefully (check column existence and pd.notna() before evaluating)
   - Comment: "Single-year companies never trigger FLAG-S02 — correct behavior, not a bug"

5. `evaluate_flags(kpis_df, financials_df, sector="healthcare", config_path=CONFIG_PATH) -> list[RedFlag]`:
   - Call `load_config()` once at the top (not per flag)
   - If config empty (missing file): return `[]`
   - Sort both DataFrames by fiscal_year ascending
   - Get latest_year row for single-KPI flags
   - Iterate sector flags from YAML; for each: check KPI exists and not NaN; evaluate Alta/Media/Baja in order; append RedFlag if triggered
   - Call `_evaluate_special_flags()` and extend flags list
   - Sort result by severity: {"Alta": 0, "Media": 1, "Baja": 2} — NOT alphabetically
   - Return sorted list

6. Module `CONFIG_PATH = Path(__file__).parent / "config" / "red_flags.yaml"`

Note: `operating_cash_flow` lives in `financials.parquet` (not kpis.parquet). For FLAG-S01, check the financials_df columns. For FLAG-S02 (consecutive losses), use `net_profit_margin` from kpis_df. The evaluate_flags() signature accepts both DataFrames.
